## Модуль 1: Основы программирования и Computer Science (~22 ч)

### 0. Что такое программа (0.5 ч)
Кнопка на экране → Python-код → C → ассемблер → машинный код (нули и единицы, идёт ток / не идёт)

### 1. Python (13 ч)

> Все подтемы лежат в папке `lesson 1 python/`. Каждая — подпапка `1.X/` с тремя файлами: теория (`.md`), код (`.py`), тесты (`test_*.py`).

Ставим бесплатный PyCharm.

- **1.1** Переменные
- **1.2** Типы данных (`int`, `float`, `str`, `bool`), операции (`+`, `-`, `*`, `/`), сравнения, `type()`, `int()`, `str()`
- **1.3** Списки, кортежи, словари, множества
- **1.4** Циклы (без вложенных — не нужны в проде, только путают)
- **1.5** `f-strings` (базово, для понимания синтаксиса)
- **1.6** Докстринги и комментарии
- **1.7** Функции
- **1.8** `import` (на примере `math`, без сторонних библиотек — просто понять механизм)
- **1.9** Декоратор (базово — чтобы понимать `@router`, `@tool`)
- **1.10** `try-except`
- **1.11** Контекстный менеджер (`with`)
- **1.12** *Опционально:* type hints (после функций)

### 2. ООП (3 ч)

## Модуль 3: AI / ML (~45.5 ч обязательных / ~50.5 ч с опциональными)

### 25. Переход: как бэкендеру зайти в AI (1 ч)
- Карта профессий в AI: кто такой AI-разработчик, ML-инженер, Data Scientist
- Почему бэкендеру проще войти в AI, чем дата-сайентисту в продакшен
- Что из бэкенд-стека переиспользуется: API, базы, очереди, контейнеры

### 26. Математика на пальцах (4 ч)
Без доказательств, чисто прикладное:
- Функция
- Градиент (ищем минимум → хорошо)
- Векторы, матрицы, операции над ними
- Скалярное произведение, softmax
- Косинусное расстояние
- Размерности и эмбеддинги — интуиция: почему вектор из 768 чисел «понимает» смысл

### 27. ML (3 ч)
- Регрессия, классификация
- Функция потерь
- Основные метрики

### 28. Neural Networks (2 ч)
Очень кратко: нейронка = несколько классических активаций + функции потерь между ними + backpropagation.

### 29. Transformer (4 ч)
- Идея attention: как модель «смотрит» на все слова сразу
- Self-attention, multi-head attention — на пальцах, без формул
- Q, K, V матрицы — зачем три проекции
- Positional encoding: почему порядок слов важен
- Encoder / decoder — кто за что отвечает

### 30. GPT (2 ч)
- Контекстное окно, промпты
- Почему GPT — это decoder-only

### 31. Данные для дообучения (2 ч)
- Откуда берутся данные: открытые датасеты, синтетика, краудсорс
- Очистка и дедупликация
- Форматы: instruction-tuning (вопрос-ответ), чаты, function calling
- Train / val / test — зачем нужен сплит

### 32. Дообучение (2 ч)
LoRA, QLoRA, дистилляция, квантизация.

### 33. Проект: вызов API модели (2 ч)
- Добавляем в проект вызов API какой-нибудь LLM
- Первые грабли (30 мин): модель вернула чушь, таймаут, rate limit — что делать

### 34. Embeddings (2 ч)
Векторизация текста, векторные БД.

### 35. RAG (теория) (2 ч)
Retrieval-Augmented Generation, чанкинг, поиск.

### 36. Проект: RAG (4 ч)
Импортируем LangChain (пока не рассказываем зачем), настраиваем RAG для `pgvector`, добавляем поиск по векторам.

### 37. LangChain (5 ч)
Цепочки, промпты, интеграции.

### 38. LangGraph (5 ч)
Графы состояний, flow агентов.

### 39. Проект: Агенты (4 ч)
- Пишем агентскую обвязку в проекте
- Добавляем эндпоинт `/agent`
- Пишем тулы для агента

#### 39.1 (опционально) LangFuse (2 ч)
Добавляем в `docker-compose`, подключаем к проекту, объясняем смысл трейсинга.

### 40. Failure Modes и Structured Output (2 ч)
- Галлюцинации: почему модель врёт и как с этим жить
- RAG-провалы: не те чанки, потеря контекста
- Агент зациклился — как дебажить
- Structured Output: JSON mode, function calling как гарантия формата
- Базовая стратегия: логгирование каждого шага → поиск места сбоя

### 41. (опционально) MCP (2 ч)
Добавляем отдельный MCP-сервер или используем реальный MCP, интегрируем как инструмент агента.

### 42. Ollama (2 ч)
Локальная модель. Заменяем API-модель на локальную.

### 43. vLLM (опционально, теория) (1 ч)
- Стандарт индустрии для production-инференса
- Чем отличается от Ollama: throughput, continuous batching, PagedAttention
- Ограничиваемся теорией — на Mac не запустится, практика требует Linux-сервера

### 44. Kubernetes (теория) (1.5 ч)
- Поды, сервисы, деплойменты, ingress — на уровне концепций
- Зачем нужна оркестрация, как выглядит кластер
- Без практики: мы не devops, вручную кластер не поднимаем

## Принципы

- **Один инкрементальный проект:** от простого роутера на `dict` → до агентов с RAG в Kubernetes
- **Всё разворачивается локально**, без платных облаков
- **opencode** включается с этапа 9 — как инструмент продуктивного разработчика, а не костыль
- Опциональные блоки помечены — обязательный трек остаётся компактным
