# Урок 0: Что такое программа

---

## Интуиция: от кнопки до тока

Представь: ты открываешь сайт, заполняешь форму и нажимаешь кнопку **«Зарегистрироваться»**. Что происходит внутри?

```
      Кнопка на сайте
            ↓
         API (HTTP)
            ↓
       Python-код
            ↓
        C (CPython)
            ↓
        Ассемблер
            ↓
       Машинный код
            ↓
      Нули и единицы
            ↓
    Идёт ток / не идёт ток
```

Разберём этот путь шаг за шагом.

---

## Слой 1: Кнопка на сайте

Вот что ты видишь в браузере:

```
┌──────────────────────────────┐
│                              │
│   Имя:    [______________]   │
│   Почта:  [______________]   │
│                              │
│   [ Зарегистрироваться ]     │  ← тыкаешь сюда
│                              │
└──────────────────────────────┘
```

Ты нажал кнопку — и как будто ничего не произошло. Но на самом деле произошло главное: **браузер отправил запрос**. Кому? Серверу. Где? По адресу в интернете. Как? Через API.

---

## Слой 2: API — «курьер» между сайтом и сервером

Кнопка дёргает API-запрос. Это означает: браузер формирует сообщение и отправляет его по интернету на сервер.

```http
POST /api/register HTTP/1.1
Host: mysite.com
Content-Type: application/json

{
  "name": "Анна",
  "email": "anna@example.com"
}
```

Перевод на человеческий:

> *«Сервер mysite.com, прими запрос. Я хочу создать нового пользователя. Вот данные: имя — Анна, почта — anna@example.com.»*

Сервер — это просто компьютер (такой же, как твой ноутбук), который всегда включён и ждёт запросов. Его задача: принять запрос, обработать и ответить.

---

## Слой 3: Python-код (то, что мы будем писать)

Сервер получает запрос и запускает наш код:

```python
@app.post("/api/register")
def register_user(name: str, email: str):
    user = {"name": name, "email": email}
    save_to_database(user)
    return {"status": "ok", "message": f"Привет, {name}!"}
```

Это понятный человеку текст. Но есть проблема: **процессор не понимает Python.** Он вообще не понимает никакой язык программирования. Ему нужно что-то гораздо более примитивное.

**Программа** — это просто набор инструкций, записанных в файл. Когда мы «запускаем программу», мы говорим компьютеру: «Прочитай эти инструкции и выполни их по порядку».

Но как процессор выполнит Python-код, если он его не понимает? Дальше начинается магия трансляции.

---

## Слой 4: CPython — переводчик с Python на байт-код

Python-код не исполняется напрямую. Программа `python` (сама написанная на языке C) читает твой `.py`-файл и превращает его в **байт-код** — промежуточное представление:

```
register_user(...)  →  [LOAD_FAST, LOAD_GLOBAL, CALL_FUNCTION, RETURN_VALUE, ...]
```

Это всё ещё не то, что понимает процессор, но уже на шаг ближе.

---

## Слой 5: C — язык системного уровня

Интерпретатор Python написан на языке C. C уже стоит вплотную к железу:

```c
printf("User registered: %s\n", name);
```

C-код компилируется (переводится) в ассемблер — команды, почти один-в-один с командами процессора.

---

## Слой 6: Ассемблер

Ассемблер — это самый низкий уровень, который ещё читается человеком (с трудом):

```asm
mov rax, 1        ; положи 1 в регистр rax
mov rdi, 1        ; файловый дескриптор (stdout)
mov rsi, message  ; адрес строки в памяти
mov rdx, 26       ; длина строки
syscall           ; попроси ядро ОС выполнить
```

Никаких `for`, классов, функций. Только элементарные действия: «положи туда», «сравни это», «прыгни вот сюда», «попроси систему».

---

## Слой 7: Машинный код — нули и единицы

Ассемблер превращается в машинный код — чистую последовательность нулей и единиц:

```
10110000 00000001   →  mov al, 1
11001101 10000000   →  int 0x80
```

Каждая комбинация бит означает конкретную операцию. Процессор на лету декодирует: «Ага, 10110000 — значит положи 1 в регистр AL».

---

## Слой 8: Транзисторы — идёт ток / не идёт ток

На физическом уровне процессор состоит из миллиардов **транзисторов** — микроскопических переключателей:

- **1** → транзистор открыт → ток идёт
- **0** → транзистор закрыт → ток не идёт

Всё, что делает компьютер, сводится к одному: **есть напряжение / нет напряжения**. Миллиарды таких переключений в секунду — и в сумме получается регистрация пользователя, видеозвонок, запуск ракеты и котики в Telegram.

---

## Аналогия: доставка пиццы

| Доставка пиццы | Компьютер |
|----------------|-----------|
| Ты нажал «Заказать» в приложении | Кнопка на сайте |
| Приложение отправило заказ на кухню | API-запрос на сервер |
| Повар прочитал заказ и начал готовить | Python-код обработал запрос |
| Рецепт: раскатать тесто, нарезать, в духовку | C/Ассемблер: элементарные инструкции |
| Духовка: нагреть до 200°C | Процессор: выполнить 10110000 |
| Газ: горит / не горит | Транзистор: ток идёт / не идёт |

На каждом шаге информация спускается на уровень ниже. Пиццерия не знает, что ты заказал «Маргариту» — ей нужна только команда «нагреть духовку». Так и процессор не знает, что ты регистрируешь пользователя — он просто выполняет миллионы нулей и единиц.

---

## Главная мысль

> **Программа** — это текст, который проходит цепочку превращений: от нажатия кнопки на сайте → через API, Python, C, ассемблер → до электрических сигналов в процессоре.

Мы пишем на самом верхнем уровне — Python и API. Всё остальное делают за нас интерпретатор, компилятор, операционная система и процессор. **Мы управляем всей этой пирамидой, сидя на её вершине.**
